# im-code-map v5.1.0 Excalidraw 시각 설계 검토

검토일: 2026-07-14

## 1. 검토 결론

v5.0의 문제는 Excalidraw 파일 형식이나 아이콘 개수가 아니라 **정보의 시각적 우선순위가 약했다는 점**이다.

기존 그림은 다음 요소가 거의 같은 강도로 보였다.

- 단계 카드와 보조 설명
- 정상 흐름과 예외 흐름
- 프레임, 링크, 상태값, 신뢰도 표시
- 개별 문서로 이동하는 내부 경로

결과적으로 정보는 존재했지만, 사람이 처음 보았을 때 어디서 시작하고 어느 방향으로 읽어야 하는지가 충분히 강하지 않았다. v5.1은 데이터 모델을 다시 축소하지 않고 **동일한 근거를 Focus용 스토리보드와 Atlas용 시스템 지도라는 서로 다른 시각 문법으로 투영**하도록 변경했다.

## 2. 외부 자료 검토와 적용 범위

다음 유형의 자료를 검토했다.

- 시스템 디자인 Excalidraw 모음
- 공식 Excalidraw Libraries와 `.excalidrawlib` 형식
- Obsidian Excalidraw 예제 vault와 ExcalidrawAutomate 패턴
- C4·DFD·flowchart에 가까운 경계·컴포넌트·흐름 표현

참고한 공개 위치는 `sources.md`에 기록했다.

외부 자료의 **레이아웃 문법과 재사용 방식**은 참고했지만, 저장소 전체 또는 개별 도형의 재배포 허가가 확인되지 않은 경우 원본 요소를 복사하지 않았다. 특히 일부 모음은 저장소 LICENSE가 확인되지 않거나 README에서 제3자·유료 컴포넌트 출처를 밝히고 있었다. 따라서 v5.1은 다음 원칙을 사용한다.

```text
공개되어 있음 ≠ 재배포 가능함
시각 패턴 참고 ≠ 도형 파일 복사
```

패키지에는 외부 로고, 이미지, 아이콘 폰트, 제3자 `.excalidrawlib`를 넣지 않았다.

## 3. Focus 디자인 변경

### 3.1 일관된 왼쪽→오른쪽 읽기

v5.0의 긴 흐름은 행마다 방향이 바뀌는 방식이 포함될 수 있었다. v5.1은 이를 제거하고 모든 의미 단계 행을 왼쪽에서 오른쪽으로 읽게 했다.

```text
PHASE 1 주문 가능성 확인  →  1 → 2 → 3 → 4
PHASE 2 재고와 결제 준비  →  5 → 6 → 7 → 8
PHASE 3 승인과 완료        →  9 → 10 → 11 → 12
```

행 사이 연결선은 카드 위를 가로지르지 않고 여백을 따라 이동한다.

### 3.2 카드 우선순위

각 중심 카드에는 다음만 첫 화면에 둔다.

- 단계 번호
- 의미 아이콘
- 업무 언어 제목
- 한두 줄 설명
- 필요한 경우 짧은 상태 변화

코드 경로, 테스트 위치, 상세 조건식, 반복되는 `PARTIAL` 표시는 연결된 문서와 Atlas로 이동한다.

### 3.3 정상 흐름과 경계 분리

중앙에는 사용자가 기대하는 정상 흐름을 배치한다. 아래에는 다음 세 종류의 패널을 별도로 둔다.

- 왜 흐름이 멈추는가
- 아직 확인되지 않은 것은 무엇인가
- 어떤 경우 Deep Atlas를 열어야 하는가

예외를 숨기지는 않되 정상 흐름과 같은 시각적 비중으로 펼치지 않는다.

### 3.4 질문 중심 탐색 버튼

그림 상단에 원시 `[[wiki/path]]` 텍스트를 노출하지 않는다. 다음과 같은 짧은 이동 버튼으로 표현한다.

- 질문 문서
- 흐름 문서
- 미확인 사항
- Deep Atlas

실제 Obsidian 링크는 요소의 `link` 속성에 보존한다.

## 4. Human Overview 변경

첫 화면은 다섯 단계의 안내 카드로 재구성했다.

```text
START HERE
→ 질문에서 시작
→ 한 흐름을 끝까지
→ 현재 이해의 경계
→ Deep Atlas 열기
```

상단은 제목과 탐색, 중앙은 읽는 길, 하단은 관련 미확인 사항과 범위 밖 영역으로 분리했다. 사용자가 문서 폴더 구조를 먼저 이해해야만 지도를 읽을 수 있는 역전 현상을 제거했다.

## 5. Deep Atlas 디자인 변경

### 5.1 조용한 시스템 경계

Atlas는 정보량이 많기 때문에 배경 경계와 lane을 낮은 대비로 표시하고, 실제 업무 노드와 조건 분기가 전면에 보이게 했다.

### 5.2 의미별 컴포넌트 카드

다음 종류는 서로 다른 색·아이콘·상단 강조선을 사용한다.

- actor
- service/process
- decision
- state transition
- event/queue
- storage/data
- external system
- risk/error
- compensation
- outcome
- domain

색상만으로 의미를 구분하지 않고 아이콘과 제목을 함께 사용한다.

### 5.3 Decision 표현 이원화

- Focus에서는 읽기 쉬운 **decision card**를 기본으로 사용한다.
- Atlas에서는 실제 분기 구조가 중요한 경우 **decision diamond**를 사용한다.

모든 판단을 마름모로 만들거나 모든 것을 사각형으로 만드는 양쪽의 전통적 고집을 피했다.

### 5.4 도메인 지도

도메인 개요는 단순한 도메인 나열이 아니라 중심 orchestration 도메인과 주변 책임 도메인을 hub-and-spoke 형태로 배치한다. 연결선에는 `query`, `command`, `event`와 확인된 계약명을 표시한다.

## 6. Master Infinite Canvas 변경

master는 자식 도면을 임의 좌표에 축소해 붙이는 방식 대신 다음 구조를 사용한다.

1. 상단의 numbered section index
2. 책임 지도
3. 주 업무 stream
4. 상태·자격 지도
5. 조회 stream
6. 취소·복구 stream

각 섹션은 충분한 크기로 합성되며 `Open child` 링크를 제공한다. child 파일이 유지보수 원본이고 master는 탐색용 통합 뷰라는 계약은 유지한다.

## 7. 자체 Stencil Library

다음 파일을 추가했다.

```text
libraries/im-code-map-architecture.excalidrawlib
```

14개의 재사용 컴포넌트를 포함한다.

1. Actor / 사용자
2. Service / 서비스
3. Decision card / 판단
4. Decision diamond / 상세 분기
5. State transition / 상태 변화
6. Event / 비동기 이벤트
7. Database / 저장소
8. Queue / 메시지 큐
9. External system / 외부 시스템
10. Risk / 미확인 경계
11. Compensation / 보상
12. Outcome / 종료 결과
13. Domain / 도메인
14. Deep Atlas link / 상세 지도 링크

모든 요소는 네이티브 Excalidraw primitive로 생성된다. 이미지 파일, 외부 SVG, 로고, 아이콘 폰트를 포함하지 않는다.

## 8. 디자인 토큰과 테마

공통 토큰은 다음 파일에서 관리한다.

```text
templates/excalidraw-design-system.json
scripts/excalidraw_design_system.py
```

지원 테마는 두 가지다.

- `clean`: 기본. 낮은 roughness, 정돈된 카드와 경계, 문서·설명용
- `whiteboard`: 같은 의미 구조를 유지하면서 Excalidraw의 손그림 질감을 사용하는 선택 테마

테마는 장식 옵션일 뿐, 노드 순서나 의미 구조를 변경하지 않는다.

## 9. 생성·검증기 변경

### 생성

- `generate_excalidraw_from_visual_model.py`: Focus storyboard, Atlas cards·lanes·domain map
- `compose_excalidraw_master.py`: curated-grid master와 섹션 index
- `generate_excalidraw_stencil_library.py`: 패키지 소유 stencil 생성
- `generate_excalidraw_automate_scripts.py`: v5.1 색상·아이콘·roughness 적용

### 렌더

`render_excalidraw_preview.py`는 다음 순서로 그린다.

```text
배경·lane
→ edge
→ 카드·도형
→ 아이콘
→ 텍스트
```

따라서 연결선이 카드의 설명문 위를 지나가는 문제를 줄였다.

### 검증

`validate_excalidraw_output.py`는 기존 의미 검증과 함께 다음을 확인한다.

- `designSystemVersion`
- Focus의 원시 wiki 경로 노출 금지
- navigation button과 step badge
- narrative boundary panel
- master의 `curated-grid`
- child link
- native vector icon
- 분기 조건, edge binding, 상태 전이, 오류·보상 구조

## 11. 남아 있는 한계

- commerce 예제는 기능 검증용 synthetic example이며 Papercompany의 실제 업무 규칙이 아니다.
- Deep Atlas의 상세 stream은 정보량이 많으므로 master 전체 화면에서 문장을 읽기보다 섹션을 확대하거나 child 파일을 여는 방식이 전제다.
- 미리보기 렌더러는 패키지가 생성하는 primitive subset을 확인하는 도구이며, Excalidraw 공식 렌더러와 픽셀 단위로 동일하다고 주장하지 않는다.
- 외부 라이브러리는 사용자가 별도로 설치할 수 있지만, 패키지에 합치려면 개별 asset의 라이선스와 귀속을 다시 확인해야 한다.

## 12. 최종 판단

v5.1은 v5의 Focus/Atlas 모델을 바꾸지 않고 **사람이 읽는 순서와 시각적 계층을 다시 설계한 minor release**다.

```text
v5.0: 무엇을 어느 깊이로 보여 줄 것인가
v5.1: 그것을 사람이 실제로 어떻게 읽게 만들 것인가
```

장식적인 아이콘 추가가 아니라, 질문→중심 흐름→경계→상세 지도의 이동 경로를 그림 자체가 안내하도록 바꾼 것이 핵심이다.
