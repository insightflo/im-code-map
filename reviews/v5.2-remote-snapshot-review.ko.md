# v5.2 원격 저장소 증거 모드 검토

## 확인된 v5.1 결함

v5.1은 정상 모드를 로컬 Git과 CodeGraph 설치 여부에 결박했다. 그러나 저장소 커넥터가 고정 커밋의 파일 내용, blob SHA, 정확한 줄 범위를 제공하는 환경에서도 CodeGraph가 없다는 이유만으로 분석을 저하 모드로 취급했다. 이는 도구 이름을 증거 품질과 혼동한 설계 결함이다.

## 수정

v5.2는 정상 증거 경로를 다음으로 분리한다.

- `local-codegraph`
- `remote-connector-snapshot`
- `hybrid`

원격 모드는 전체 commit SHA, 파일별 blob SHA, 줄 범위, 증거 분류, 교차 검증, 미확인 범위를 기록해야 한다. 검색 결과나 파일명만으로는 정상 모드가 되지 않는다.

## 안전장치

`validate_repository_snapshot.py`는 다음을 검사한다.

- 모든 파일이 같은 40자 commit SHA에 고정됐는가
- 파일마다 40자 blob SHA가 있는가
- 줄 범위가 유효한가
- 파일 수와 증거 분류 집계가 일치하는가
- route/service/test/schema 핵심 증거가 있는가
- 중요 주장이 둘 이상의 증거 분류로 교차 확인됐는가
- 불완전한 snapshot이 PASS로 표시되지 않았는가

## 결과

원격 분석은 더 이상 자동 저하 모드가 아니다. 다만 가져오지 않은 호출 관계나 런타임 동작을 추측하는 순간 다시 저하되므로, `coverage.json`의 미확인 경계를 필수로 유지한다.
